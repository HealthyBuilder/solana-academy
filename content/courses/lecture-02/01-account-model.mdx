## 账户模型

- Solana 的账户是一个存储单元，和 Linux 的文件很类似。
- 用地址表示：Ed25519 公钥的 Base58 编码格式字符串，如 `14grJpemFaf88c8tiVb77W7TYg2W3ir6pfkKz3YjhhZ5`。
- 最大存 10M 数据，空间越大存储费用越高；只有 Owner 才可以修改账户数据或减少余额。

### 账户模型 → 编程模型

- 程序和数据是各自独立的账户（用 `executable` 区分），代码和数据分离。
- 程序是数据账户的 Owner，负责管理数据账户。
- 程序是无状态的，这为大规模并发执行提供了条件。

### PDA（Program Derived Addresses）账户

- 专门用来保存程序数据的账户，没有私钥。
- 曲线外生成的账户：根据 程序（owner）+ 种子 + bump 生成。

## Token on Solana

代币是数字资产，代表区块链网络上的所有权、访问权限或价值。可以将它们视为完全以数字形式存在的可编程证书：它们可以代表从货币和所有权股份到投票权和访问通行证的任何事物。

从最基本的层面来看，代币是区块链账本中的一条记录，表示“此地址拥有 X 数量的此资产”。与需要实物证书或集中式数据库来跟踪所有权的传统资产不同，代币利用区块链技术创建了防篡改、透明的所有权记录。

### 为什么代币很重要

代币实现了可编程所有权：任何人都可以创建具有自定义规则的数字资产，这些规则规定了代币如何转移、授予哪些权利或如何与其他系统交互。这为自动分红、条件转移或基于预定义条件自动执行的复杂金融工具打开了大门。

关键的创新在于，代币可以在全球范围内即时转移，无需中介，具有透明性（任何人都可以验证所有权），并且可以通过复杂的逻辑进行编程，同时保持底层区块链的安全性保障。

### 不同类型的代币

按**用途**可分为：

- **实用型代币**：提供对产品或服务的访问，例如平台访问权限、折扣费用或特殊功能。
- **治理型代币**：赋予持有者在去中心化组织中的投票权。
- **证券型代币**：以数字形式代表对现实世界资产（公司股份、房地产、商品）的所有权。
- **稳定币**：旨在保持稳定价值，通常与某种货币挂钩，例如美元（USDC、USDT）。
- **Meme / 社区代币**：主要用于投机、社区建设或娱乐价值。

按**技术特性**可分为同质化与非同质化：

- **同质化代币（FT）**：可互换，由多个相同单位组成，可分为小数单位（例如 0.5 个代币）。
- **非同质化代币（NFT）**：独特且不可互换，每个代币独一无二、不可分割。

从 Token Program 实现角度看，FT 与 NFT 是一样的，靠数量上的约束来体现 NFT 特性：

- 供应量为 1（独一无二）
- 小数位为 0（不可分割）
- 禁用 mint authority（防止再铸造）

Token Program 本身无法强制执行这些特性。因此像 MPL-token-metadata 这样的程序不仅提供元数据实现，还提供强制执行这些约束的实现，从而可以轻松创建 NFT。

### 与以太坊的区别

在以太坊中，每种代币都需要部署一个完整的智能合约，包含所有代币逻辑、状态管理和转移规则。尽管大多数代币遵循 ERC-20 标准，但每种代币都是一个带有自定义代码的独立程序，合约地址成为代币标识符，所有余额存储在该合约的状态中。

在 Solana 上，由于程序（逻辑）与数据分离，所有与代币相关的逻辑都由 **SPL Token Program** 和 **Token2022 Program** 处理：这是统一的原生代币框架，定义了所有代币的创建、管理和转移方式，确保一致性和互操作性。代币的唯一性由 **Mint 账户**确定，它相当于代币在 Solana 上的“出生证明”。

## Token 相关的账户

Token 主要涉及 3 个账户：**Token Program**、**Mint Account**、**ATA Account（Token Account）**。

### Token Program

SPL Token 程序或 Token 2022 程序，包含所有代币逻辑、状态管理与转移规则。主要功能：

- **初始化账户**：正确的所有者（Token Program）、空间和 lamports。
- **Mint To**：增加代币总供应量，创建新代币并存入指定账户（仅 mint 权限持有者可执行）。
- **Burn**：通过从流通中移除代币来永久销毁，减少总供应量。
- **转账**：将代币从一个账户转移到另一个账户。
- **授权（Approve）**：授予代理权限，允许其代表账户所有者转移特定数量的代币。
- **Revoke**：移除代理权限，将完全控制权归还账户所有者。
- **Close Account**：关闭一个余额为零的代币账户，并将其剩余 SOL 租金转移到目标账户。

### Mint 账户

代币通过由 Token Program 拥有的 Mint 账户地址唯一标识。Mint 账户存储：

- **供应量（supply）**：代币的总供应量
- **小数位数（decimals）**：代币的小数精度
- **Mint 权限（mint_authority）**：被授权增发代币的账户
- **冻结权限（freeze_authority）**：被授权冻结 Token 账户的账户

### 元数据

代币的名称、符号和图像称为 **Metadata**。Mint 账户在原生形式下只是一个 32 字节的公钥，没有人类可读信息。原生 SPL Token 程序无法直接设置元数据，因此 Metaplex 开发了 MPL-token-metadata 程序为代币关联元数据。使用 Token Extensions 与 Token2022 程序后，Metadata 扩展允许将元数据直接嵌入 mint 账户，无需依赖外部程序。

示例：https://solscan.io/token/EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v

### Associated Token Account（关联 Token 账户）

ATA 是一个特定的 Token Account，用于跟踪每个代币单位的个人所有权。它存储：

- **Mint**：该账户持有哪种代币
- **Owner**：被授权转移代币的账户
- **Amount**：当前持有的代币数量

ATA 的地址根据所有者地址和 mint 账户地址派生（Associated Token Program 的 PDA）：

```text
ATA = findProgramAddress(
  [wallet, token_program_id, mint],
  associated_token_program_id
)
```

## Demo：SPL-Token CLI

**创建 Token**（其实是创建 mint 账户，保存 decimals / supply / mint_authority / freeze_authority）：

```bash
# 生成一个以 Lbc 开头的靓号 mint 地址
solana-keygen grind --starts-with Lbc:1
# 用指定 keypair 创建 token（默认 decimals=9，可用 --decimals 6 调整）
spl-token create-token mint_keypair.json
```

**发行 Token**（需先创建 ATA 账户）：

```bash
spl-token create-account <MINT_ADDRESS>
spl-token mint <MINT_ADDRESS> <TOKEN_AMOUNT>
spl-token balance <MINT_ADDRESS>
spl-token supply <MINT_ADDRESS>
spl-token account-info <MINT_ADDRESS>
```

**转账 Token**（同样需要目标的 ATA，或加 `--fund-recipient`）：

```bash
spl-token transfer <MINT_ADDRESS> <TOKEN_AMOUNT> <RECIPIENT_WALLET_OR_TOKEN_ACCOUNT>
spl-token accounts
```

**关闭 ATA 账户**（余额为 0 时可回收账户资金）：

```bash
spl-token close <MINT_ADDRESS>
# 若有余额，需要先清空
spl-token burn <TOKEN_ACCOUNT> <AMOUNT>
```

**创建 NFT**（NFT 是 decimals 为 0 的特殊 token，每个 NFT 需要独立的 mint 账户）：

```bash
spl-token create-token --decimals 0
```

**Token2022**（在 SPL Token 指令基础上添加扩展模块）：

```bash
spl-token create-token --enable-metadata --decimals 0 --program-2022
spl-token initialize-metadata <MINT> "NAME" "Symbol" "https://example.com/nft-metadata.json"
```

## 使用 web3.js 铸造 SPL 代币

Solana web3.js 是一个流行而简洁的 JavaScript 客户端库，让开发者便捷地与 Solana 区块链交互，与以太坊的 ethers.js 类似。

```bash
npm install @solana/web3.js@1.98.0
```

导入所需库：

```typescript
const solanaWeb3 = require("@solana/web3.js");
const splToken = require("@solana/spl-token");
const bs58 = require("bs58");
```

- `@solana/web3.js` — Solana 区块链的主要 JS API
- `@solana/spl-token` — SPL Token 操作库（铸币、代币账户等）
- `bs58` — Base58 编码／解码（如私钥）

连接到 Solana（这里使用官方 Devnet 节点）：

```typescript
const connection = new solanaWeb3.Connection(
  "https://api.devnet.solana.com",
  { wsEndpoint: "wss://api.devnet.solana.com" }
);
```

主函数与加载钱包：

```typescript
async function main() {}

const walletKeyPair = solanaWeb3.Keypair.fromSecretKey(
  new Uint8Array(bs58.decode(process.env.PRIVATE_KEY))
);

let balance = await connection.getBalance(walletKeyPair.publicKey);
console.log(balance / solanaWeb3.LAMPORTS_PER_SOL);
```

### 1. 铸造 SPL Token

```typescript
const mint = await splToken.createMint(
  connection,
  walletKeyPair,
  walletKeyPair.publicKey,
  null, // freeze authority
  9, // decimals
  undefined,
  {},
  splToken.TOKEN_PROGRAM_ID
);

const tokenAccount = await splToken.getOrCreateAssociatedTokenAccount(
  connection,
  walletKeyPair,
  mint,
  walletKeyPair.publicKey
);

await splToken.mintTo(
  connection,
  walletKeyPair,
  mint,
  tokenAccount.address,
  walletKeyPair.publicKey,
  1000000000000
);
```

### 2. 发送 SOL 交易

```typescript
const secondWalletKeyPair = solanaWeb3.Keypair.generate();

const transaction = new solanaWeb3.Transaction().add(
  solanaWeb3.SystemProgram.transfer({
    fromPubkey: walletKeyPair.publicKey,
    toPubkey: secondWalletKeyPair.publicKey,
    lamports: solanaWeb3.LAMPORTS_PER_SOL * 0.001,
  })
);

const signature = await solanaWeb3.sendAndConfirmTransaction(
  connection,
  transaction,
  [walletKeyPair]
);
```

### 3. 质押 / 委托 SOL

```typescript
const stakeAccount = solanaWeb3.Keypair.generate();
const createStakeAccountInstruction = solanaWeb3.StakeProgram.createAccount({
  fromPubkey: walletKeyPair.publicKey,
  stakePubkey: stakeAccount.publicKey,
  authorized: new solanaWeb3.Authorized(
    walletKeyPair.publicKey,
    walletKeyPair.publicKey
  ),
  lamports: solanaWeb3.LAMPORTS_PER_SOL * 0.02,
});

const delegateInstruction = solanaWeb3.StakeProgram.delegate({
  stakePubkey: stakeAccount.publicKey,
  authorizedPubkey: walletKeyPair.publicKey,
  votePubkey: new solanaWeb3.PublicKey("{validator public key}"),
});
```

至此，我们用 solana-web3.js 完成了三个关键操作：✅ 铸造 SPL 代币、✅ 发送 SOL 交易、✅ 委托（Stake）SOL。做完这些，你就具备了 Solana 核心开发流程的基础技能。
